LOGIN PAGE

Set the Faculty Login page as the default landing page when the application opens.

Navigation Order:
1. Faculty Login (Default)
2. Student Login
3. Admin Login

Create three separate login pages:

• Faculty Login (Default First Page)
• Student Login (Second Page)
• Admin Login (Third Page)

When the application starts:
- Automatically open the Faculty Login page.
- Users can switch between Faculty Login, Student Login, and Admin Login using tabs or buttons.
- If the user is not logged in, always redirect them to the Faculty Login page.

FACULTY LOGIN

Fields:
- Email or Faculty ID
- Password
- Remember Me
- Forgot Password
- Login Button
- Register as Faculty Button

After successful login:
- Redirect to Faculty Dashboard.

STUDENT LOGIN

Fields:
- Email or Register Number
- Password
- Remember Me
- Forgot Password
- Login Button
- Register as Student Button

After successful login:
- Redirect to Student Dashboard.

ADMIN LOGIN

Fields:
- Username
- Password
- Login Button

Default Admin Account

Username:
admin

Password:
Admin@123

After successful login:
- Redirect to Admin Dashboard.

REGISTRATION

Create separate registration pages for Faculty and Students.

Registration should NOT require admin approval.

After successful registration:
- Save all information in the database.
- Users can log in immediately.
- Admin can view all registered users.

VALIDATION

- Prevent duplicate Register Numbers.
- Prevent duplicate Faculty IDs.
- Prevent duplicate Email IDs.
- Password must contain at least 8 characters.
- Show validation messages for incorrect input.

UI REQUIREMENTS

- Modern professional college management design.
- Faculty Login opens by default.
- Student Login is the second option.
- Admin Login is the third option.
- Responsive for mobile, tablet, and desktop.
- Support both Light Mode and Dark Mode.
- Smooth animations.
- College logo placeholder.
- Professional icons and dashboard cards.
